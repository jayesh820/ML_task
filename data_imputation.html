Overview

Imputation replaces missing values with plausible entries to enable robust analytics and modeling. The right technique depends on the data type, the mechanism of missingness, and downstream goals (prediction vs. statistical inference).

Tip: Start with an audit—quantify percent missing per column, visualize patterns, and understand collection processes. This informs both method choice and risk.

Missingness Types (Why data is missing)

MCAR

Missing Completely At Random: missingness independent of observed/unobserved data. Simple methods often acceptable.

MAR

Missing At Random: depends on observed variables. Multivariate/model-based approaches are preferred.

MNAR

Missing Not At Random: depends on unobserved values. May require modeling the missingness process or domain rules.

Core Techniques

1) Simple Fills

  • Mean/Median (numeric): fast baseline; median is robust to outliers.
  • Mode (categorical): preserves the most frequent category.
  • Constant/“Unknown”: explicit placeholder for categorical text or sentinel numeric.
  • Min/Max boundary: use known measurement bounds when devices saturate.
Caution: Simple fills can bias distributions and shrink variance—monitor impact on correlations and model performance.

2) Nearest-Neighbor

  • KNN imputation: use similar rows to fill values (mean for numeric, mode for categorical). Sensitive to scaling and k.

3) Model-Based

  • Regression/Classification: predict missing values using other features.
  • Iterative Multivariate (MICE): model each feature with missing data in rounds, capturing complex relationships.
  • Multiple Imputation: generate multiple completed datasets and pool results for valid inference.

4) Survey/Official Stats

  • Hot deck: borrow from similar records within the same dataset.
  • Cold deck: borrow from an external reference dataset.

Time-Series Methods

  • Forward/Backward fill: propagate adjacent values when local continuity is expected.
  • Linear interpolation: bridge gaps assuming smooth change.
  • Rolling averages: smooth noise; beware of lag/phase distortion.
  • Seasonal-aware: impute using seasonal neighbors (e.g., same weekday/hour).

Advanced Methods

  • Tree-based (e.g., MissForest): robust to nonlinearity and mixed data types.
  • Matrix completion: factorization for high-dimensional sparse tables.
  • Deep learning (e.g., VAEs/GAIN, specialized temporal models): strong on complex structure; requires data and validation.

How to Choose

  1. Diagnose missingness: MCAR/MAR/MNAR and percent missing by column/row.
  2. Match to data shape: tabular vs. time series; categorical vs. numeric; correlations.
  3. Set objective: prediction vs. inference—prefer multiple imputation for inference.
  4. Start simple, escalate: baseline with simple fills; iterate to KNN/MICE; consider advanced if needed.
  5. Validate: quantify error via masking, distribution checks, correlation integrity, and downstream model impact.

Validation & Quality Checks

  • Mask-and-score: hide known values and measure MAE/RMSE after imputation.
  • Distribution integrity: compare histograms/CDFs; use KS tests for shifts.
  • Relationships: correlation/covariance preservation; feature-target drift.
  • Uncertainty: multiple imputation to propagate imputation variance in standard errors.
  • Business rules: enforce ranges, monotonicity, or domain constraints.

Code Examples

Python (scikit-learn, pandas)

import pandas as pd
from sklearn.impute import SimpleImputer, KNNImputer
from sklearn.experimental import enable_iterative_imputer  # noqa: F401
from sklearn.impute import IterativeImputer

df = pd.read_csv("data.csv")

# 1) Simple imputers
num_cols = df.select_dtypes(include=["number"]).columns
cat_cols = df.select_dtypes(exclude=["number"]).columns

df[num_cols] = SimpleImputer(strategy="median").fit_transform(df[num_cols])
df[cat_cols] = SimpleImputer(strategy="most_frequent").fit_transform(df[cat_cols])

# 2) KNN imputation (numeric only)
knn = KNNImputer(n_neighbors=5, weights="distance")
df[num_cols] = knn.fit_transform(df[num_cols])

# 3) Iterative multivariate (MICE-like)
mice = IterativeImputer(random_state=42, max_iter=10, sample_posterior=True)
df[num_cols] = mice.fit_transform(df[num_cols])

# 4) Time series forward/backward fill
ts = pd.read_csv("timeseries.csv", parse_dates=["timestamp"]).sort_values("timestamp").set_index("timestamp")
ts["value_ffill"] = ts["value"].ffill().bfill()
ts["value_interp"] = ts["value"].interpolate(method="time")

R (mice, VIM)

library(mice)
library(VIM)

df <- read.csv("data.csv")

# 1) Quick simple imputation
df$age[is.na(df$age)] <- median(df$age, na.rm=TRUE)

# 2) MICE
imp <- mice(df, m=5, maxit=10, method='pmm', seed=123)
completed <- complete(imp, action="long")

# 3) KNN (VIM)
df_knn <- kNN(df, k=5)

FAQ

Does imputation always help?

Imputation enables models to train but can introduce bias. Validate rigorously; sometimes modeling missingness indicators helps.

What if data is MNAR?

Consider explicit modeling of missingness, use domain rules, or sensitivity analyses; simple fills may be misleading.

Should I scale before KNN?

Yes. Standardize numeric features to ensure distance metrics are meaningful.
